| Test Case                         | Input                                             | Expected Result                                                                               | Why this Test is Useful                                  |
| --------------------------------- | ------------------------------------------------- | --------------------------------------------------------------------------------------------- | -------------------------------------------------------- |
| **1. Verify existing order**      | `GET /api/orders/ORD-1001`                        | HTTP **200 OK** and response contains `status = PAID`                                         | Verifies the API returns correct data for a valid order. |
| **2. Verify non-existing order**  | `GET /api/orders/ORD-9999`                        | HTTP **404 Not Found**                                                                        | Ensures the API handles invalid order IDs correctly.     |
| **3. Verify invalid order ID**    | `GET /api/orders/@@@`                             | HTTP **400 Bad Request**                                                                      | Checks that the API validates invalid input.             |
| **4. Verify response fields**     | `GET /api/orders/ORD-1001`                        | Response contains `order_id`, `customer_id`, `amount`, `currency`, `status`, and `created_at` | Confirms the response structure is correct.              |
| **5. Verify unauthorized access** | `GET /api/orders/ORD-1001` without authentication | HTTP **401 Unauthorized**                                                                     | Ensures only authorized users can access order details.  |
